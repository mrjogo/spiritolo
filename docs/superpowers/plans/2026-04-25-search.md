# Search Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a basic search box to the web UI that filters the recipe list as the user types, with the query mirrored to the URL. Server-side via two `pg_trgm` GIN indexes plus chained `.or()` filters from supabase-js — no schema or view changes.

**Architecture:** A single `<input>` above the existing list. URL `?q=` is the source of truth; typing → 250ms debounced `setSearchParams(..., { replace: true })` → `useEffect` fires fetch. A pure `buildSearchFilters(q)` function in `web/src/searchQuery.ts` translates the query into `name.ilike.*term*,jsonld->>recipeIngredient.ilike.*term*` filter strings, one per whitespace-separated term ≥ 3 chars. The component chains one `.or()` per term (PostgREST AND's them).

**Tech Stack:** PostgreSQL `pg_trgm` extension, Supabase migrations, Supabase JS client, React 19, react-router-dom v7, Vite, Vitest, `@testing-library/react`, `@testing-library/user-event`.

**Reference spec:** `docs/superpowers/specs/2026-04-25-search-design.md`

**Working directory:** This worktree at `.worktrees/search`. All commands assume you are running in the devcontainer from the worktree root unless noted otherwise.

**Devcontainer / Supabase note:** Supabase runs on the Mac host. From inside the devcontainer, apply migrations using the IPv4 host address — see Task 1.

```ts
function stripEdges(term: string): string {
```

```ts
type OrChain = {
  or: ReturnType<typeof vi.fn>;
  order: ReturnType<typeof vi.fn>;
};

function makeChain(rows: Row[], count: number, error: unknown = null) {
  const range = vi.fn().mockResolvedValue({ data: rows, count, error });
  const order = vi.fn(() => ({ range }));
  const or: OrChain['or'] = vi.fn();
  or.mockReturnValue({ or, order });
  const select = vi.fn(() => ({ or, order }));
  (supabase.from as unknown as ReturnType<typeof vi.fn>).mockReturnValue({ select });
  return { range, order, or, select };
}

function mockRangeResponse(rows: Row[], count: number, error: unknown = null) {
  return makeChain(rows, count, error);
}

function mockRejection(message: string) {
  makeChain([], 0, { message });
}
```

Edit `web/src/pages/RecipeList.tsx`. Add a debounced effect that watches `inputValue`. After 250ms of stillness, call `setSearchParams(...)` with `replace: true`. Reset `page` to 1 whenever `q` changes.

```tsx
const [params, setSearchParams] = useSearchParams();
const page = Math.max(1, parseInt(params.get('page') ?? '1', 10) || 1);
const q = params.get('q') ?? '';
const [inputValue, setInputValue] = useState(q);

// Keep input in sync if URL changes externally (back/forward navigation).
useEffect(() => {
  setInputValue(q);
}, [q]);

// Debounce inputValue → URL.
useEffect(() => {
  if (inputValue === q) return; // no-op when already in sync
  const handle = setTimeout(() => {
    const next = new URLSearchParams(params);
    if (inputValue === '') next.delete('q');
    else next.set('q', inputValue);
    next.set('page', '1');
    setSearchParams(next, { replace: true });
  }, 250);
  return () => clearTimeout(handle);
}, [inputValue, q, params, setSearchParams]);
```
